## src ディレクトリ

最終更新日: 2026年2月1日

このプロジェクトのソースコードが入っているトップレベルディレクトリです。

### 📚 マイコン初心者の方へ

マイコン（マイクロコントローラー）のプログラミングでは、機能ごとにファイルを分けて整理することがとても重要です。
なぜなら、1つのファイルに全てのコードを書くと：
- 🔍 **どこに何が書いてあるか分からなくなる**
- 🐛 **バグを見つけにくい**
- 👥 **複数人で開発しづらい**
- 🔄 **コードの再利用ができない**

このプロジェクトでは、**役割ごとにフォルダを分けて管理**しています。

### 📂 ディレクトリ構成と役割

#### `App/` - アプリケーション層
画面ごとの機能（情報表示、ロック画面、モーター制御など）を格納しています。
- **何のため？**: ユーザーに見える「画面」や「機能」を1つずつ独立させるため
- **初心者向けヒント**: 新しい機能を追加したい時は、ここに新しいフォルダを作ります

#### `system/` - システム基盤
マイコン全体の初期化や共通処理を置く場所です。
- **何のため？**: アプリを動かす「土台」となる処理（起動処理、タッチ管理など）
- **初心者向けヒント**: 起動時に1回だけ実行する処理などはここに書きます

#### `timer/` - タイマー管理
一定時間ごとに処理を実行するための仕組みです。
- **何のため？**: 「10msごとに」「1秒ごとに」など、定期的な処理を正確に実行するため
- **初心者向けヒント**: LEDの点滅や定期的なセンサー読み取りなどに使います

#### `UI/` - ユーザーインタフェース部品
ボタン、スライダー、スイッチなど、画面に表示する「部品」の集まりです。
- **何のため？**: 画面部品を使い回せるようにして、開発を楽にするため
- **初心者向けヒント**: 既存の部品を使えば、ボタンやスライダーを1から作る必要がありません

### 🎯 ファイル構成の基本ルール

このプロジェクトでは、各機能は通常 **2つのファイル** で構成されています：

1. **`.h` ファイル（ヘッダファイル）**
   - クラスや関数の「設計図」を書く場所
   - 「こんな関数がありますよ」という宣言だけを書く
   - 他のファイルから `#include` して使う

2. **`.cpp` ファイル（実装ファイル）**
   - 実際の処理（中身）を書く場所
   - `.h` で宣言した関数の具体的な動作を実装する

**なぜ2つに分けるの？**
- コンパイル時間が短くなる（変更した部分だけ再コンパイルできる）
- 他の人が使い方を理解しやすい（.hを見れば何ができるか分かる）
- 複数ファイルから同じ機能を使える

各サブディレクトリの README を読んで、具体的な使い方や関数の説明を確認してください。

4. `timer/` - 定期処理の仕組み

困ったことがあれば、各フォルダの README に詳しい説明があります！
