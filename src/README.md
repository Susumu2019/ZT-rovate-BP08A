
## src ディレクトリ

最終更新日: 2026年2月16日  
対応バージョン: v1.0.0

このディレクトリはプロジェクトのメインソースコードを格納します。

### 構成と役割

- `App/` : アプリケーション層（各画面・機能ごとに分割）
- `system/` : システム基盤（初期化・共通処理・タッチ管理）
- `timer/` : タイマー管理（周期処理・割り込み）
- `UI/` : UI部品（ボタン・スライダー・スイッチ等）

### ファイル構成の基本ルール

- `.h` ファイル（ヘッダ）: クラスや関数の宣言・設計図
- `.cpp` ファイル（実装）: 実際の処理内容

### 追加・カスタマイズ時の注意

- 新規アプリ追加時は `AppTemplete/` を参考に
- グローバル変数や初期化処理は `system/` 配下にまとめる
- UI部品の再利用推奨

**なぜ2つに分けるの？**
- コンパイル時間が短くなる（変更した部分だけ再コンパイルできる）
- 他の人が使い方を理解しやすい（.hを見れば何ができるか分かる）
- 複数ファイルから同じ機能を使える
